📡 团队沟通与协作方式

沟通形式 用途 频率 工具
Issues 任务分配、问题追踪、决策讨论 持续 GitHub Issues
Discussions 知识共享、架构讨论、异步协作 持续 GitHub Discussions
Pull Requests 代码/文档变更、同行评审 按需 GitHub PRs
周度同步 进度汇报、阻塞识别、决策确认 每周 1 次（可选 30 min） Zoom / Slack
决策日志 记录战略选择与取舍 实时 decision_log.md

核心原则： 异步优先。所有信息应该在 GitHub Issues/Discussions 中可见，便于团队成员随时了解进展，不依赖实时会议。
